# `typeof` оператор в JavaScript

Оператор `typeof` возвращает строку, указывающую тип операнда.

---

## ⚠️ Опасности использования `typeof`

Главный подвох `typeof` заключается в том, что он некорректно определяет `null` как "object" (это признанная ошибка языка) и возвращает "object" для массивов (так как массивы являются объектами).

Также, `typeof NaN` вернёт "number", хотя `NaN` означает "не число".

---

## Основные особенности `typeof`, которые могут ввести в заблуждение

### `typeof null`

Возвращает "object", что является исторической ошибкой в JavaScript и не подлежит исправлению из-за совместимости. На самом деле `null` — это примитивное значение, а не объект.

### `typeof []`

Возвращает "object". Хотя это правильно по стандарту, это может быть не так очевидно, так как `typeof` не различает массивы и другие объекты.

### `typeof NaN`

Возвращает "number". Значение `NaN` (Not-a-Number) является числовым типом данных, но означает "не число".

### `typeof new Number(42)`

Возвращает "object". Если вы создаёте объект-обёртку для числа (например, `new Number()`), то `typeof` вернёт "object", а не "number".

---

## Как обойти эти подвохи

- **Для null:**

  ```js
  if (myVar === null) { ... }
  ```

- **Для массивов:**

  ```js
  Array.isArray(myVar);
  ```

  или

  ```js
  myVar instanceof Array;
  ```

- **Для проверки NaN:**

  ```js
  Number.isNaN(value);
  ```

- **Для объектов-обёрток:**

  ```js
  Number(new Number(42));
  ```
